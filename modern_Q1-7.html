Q1. List out the feature of HTML 5.

1. New semantic elements such as (header), (footer), (nav), (article), (section), and (aside) to provide better structure to web pages.

2. Support for audio and video elements, allowing for easy embedding of media content without the need for third-party plugins.

3. Improved form elements with new input types like email, date, number, and range for better user input validation.

4. Canvas element for drawing graphics and animations using JavaScript.

5. Offline storage capabilities through local storage and session storage.

6. Geolocation support for accessing the user's location through the browser.

7. Web workers for running scripts in the background to improve performance.

8. Improved error handling and APIs for better interactivity and responsiveness.

9. Improved support for mobile devices with features like touch events and device orientation.

10. Support for scalable vector graphics (SVG) and MathML for better rendering of graphics and mathematical equations.



Q2. What are HTML Entites? List out 5 commonly used HTML entities.

HTML entities are special characters that are represented by a specific code in HTML. They are used to display characters that have special meanings in HTML, such as <, >, &, ", and '.

Here are 5 commonly used HTML entities:

1. <- represents the less than symbol (<)

2. > - represents the greater than symbol (>)

3. & - represents the ampersand symbol (&)

4. " - represents the double quotation mark (")

5. ' - represents the apostrophe or single quotation mark (')



Q3. What is web accessibility? List some of the assistive device which play a major role in providing accessibility.

Web accessibility refers to the practice of ensuring that websites and web applications are usable by people with disabilities. This includes making sure that people with disabilities can perceive, understand, navigate, and interact with websites effectively.


Some of the assistive devices that play a major role in providing accessibility for people with disabilities include

1. Screen readers: Screen readers are software programs that read out the text on a screen to a user who is blind or visually impaired. They convert text into speech or braille output, allowing users to navigate websites and interact with content.

2. Magnifiers: Magnification software helps users with low vision by enlarging text and images on the screen. This makes content easier to read and interact with for people with visual impairments.

3. Voice recognition software: Voice recognition software allows users to control a computer or device using voice commands. This is particularly helpful for people with mobility impairments who may have difficulty using a keyboard or mouse.

4. Alternative input devices: Alternative input devices such as sip-and-puff devices, head pointers, and eye-tracking systems allow users with physical disabilities to interact with websites and applications using alternative methods.

5. Braille displays: Braille displays provide tactile output of digital content for users who are blind or visually impaired. They convert text into braille characters that can be read by touch.



Q4. List any 3 ways which help us in improving the accessibility of HTML.

1. Semantic HTML: Using semantic HTML elements such as (header), (nav), (main), (footer), (article), (section), and (aside) helps improve accessibility by providing clear structure and meaning to the content. Screen readers and other assistive technologies rely on semantic HTML to properly interpret and present the content to users with disabilities.

2. Alternative text for images: Adding descriptive alt text to images using the alt attribute in the (img) tag ensures that users who are visually impaired or using screen readers can understand the purpose and context of the images on the website. This is crucial for providing a comprehensive and inclusive experience for all users.

3. Keyboard accessibility: Ensuring that all interactive elements on a website can be accessed and operated using only a keyboard is essential for users who cannot use a mouse or other pointing device. This includes providing keyboard focus indicators, ensuring logical tab order, and allowing users to navigate and interact with all elements using keyboard shortcuts or commands.



Q5. Write a short note on the tab index.

The tabindex attribute is used in HTML to specify the order in which elements receive focus when navigating through a webpage using the keyboard. By setting the tabindex value of an element, developers can control the sequence in which users can navigate through interactive elements on a webpage using the "Tab" key.
The tabindex attribute accepts numerical values, with elements having a lower tabindex value receiving focus first. A tabindex value of 0 indicates that an element should be included in the default tab order based on its position in the HTML document. Negative values can also be used to exclude elements from the tab order.
It is important to use tabindex judiciously and ensure that the tab order follows a logical and intuitive sequence for users. Overriding the default tab order can sometimes lead to confusing or unpredictable navigation experiences for users, so it is recommended to use tabindex sparingly and in conjunction with other accessibility best practices to create a more inclusive user experience.



Q6. List any 5 semantic tags in HTML along with their descriptions.

Header

The (header) tag represents the introductory content or a group of navigational links at the top of a webpage. It typically contains headings, logos, navigation menus, search bars, and other elements that provide context or aid in website navigation.

main

The (main) tag defines the main content area of a webpage. It should contain the primary content of the page, such as articles, blog posts, product listings, or any other central information that is unique to that specific webpage.

nav

The (nav) tag is used to define a section of a webpage that contains navigation links. It is typically used to group together links that lead to different parts of the website or external pages, helping users easily navigate through the site.

article

The (article) tag is used to define a self-contained piece of content within a webpage. It represents content that is independently distributable or reusable, such as blog posts, news articles, forum posts, or user comments.

footer

The (footer) tag represents the footer section of a webpage, typically located at the bottom. It often contains copyright information, contact details, sitemap links, social media icons, and other elements that provide additional information about the website or help users navigate away from the page.



Q7. What are the benefits of using semantic tags in our webpage.

1. Improved Accessibility: Semantic tags provide structure and meaning to the content, making it easier for screen readers and other assistive technologies to interpret and present the information to users with disabilities. This helps ensure that all users, regardless of their abilities, can access and understand the content.

2. Better SEO: Search engines like Google and Bing rely on semantic HTML to understand the content and context of a webpage. By using semantic tags appropriately, you can help search engines better index and rank your content, potentially improving your website's visibility in search results.

3. Enhanced User Experience: Semantic tags make it easier for users to navigate and interact with your website. Clear and well-defined sections like headers, navigation menus, main content areas, and footers help users quickly find the information they are looking for, leading to a more positive user experience.

4. Consistent Styling: Semantic tags provide a standardized way to structure web content, making it easier to apply consistent styling across different pages of a website. This can help maintain a cohesive design and branding throughout the site.

5. Future Compatibility: Semantic HTML is designed to be forward-compatible with new technologies and devices. By using semantic tags, you can ensure that your website remains accessible and functional across various platforms and devices, including mobile devices, voice assistants, and emerging technologies.
